1. introduction Link to heading

Active Directory (AD) is a tool for network administrators, enabling the creation and management of domains, users and objects within a corporate network. Through AD, it is possible to connect all machines to a DC (Domain Controler) server. to a DC (Domain Controler) server which authenticates users using the kerberos protocol.

For example, an administrator can create a group of users within the corp.local domain and grant them specific privileges to access folders or internal systems. As the network grows, Active Directory makes it easier to organize large volumes of users and devices into logical groups, ensuring fine-grained control over permissions and access.

Active Directory is organized as follows:

  1. Domain
  2. Tree
  3. Forest

2.1 Domain Link to heading

A domain is a logical grouping of objects (users, computers, printers, etc.) that share the same database. All resources within a domain follow the rules established by the administrator.

Example: The corp.local domain can contain users such as joao@corp.local, ana@corp.local and devices such as servidor1.corp.local.

2.2 Tree Link to heading

A tree is a collection of domains that share a continuous namespace. Within the tree, the domains have hierarchical and trust relationships with each other.

Example: Within the corp.local tree, we can have secondary domains such as:

  • ti.corp.local (for the IT team)
  • finance.corp.local (for the finance department)
  • marketing.corp.local (for the marketing team)

2.3 Forest Link to heading

The forest represents the highest level of the Active Directory hierarchy. It is made up of multiple trees that share a common security scheme and configuration, but can have distinct and independent domains.

Example: A multinational company may have a forest containing different trees:

  • corp.local (for the head office)
  • europe.corp.local (for operations in Europe)
  • asia.corp.local (for operations in Asia)

Reconnaissance Link to heading

in a scenario in which an attacker manages to compromise an entry point within the target network either by exploiting an open Internet service, obtaining vpn passwords or by phishing the internal reconnaissance process is initiated, the objective in question is to compromise the DC a.k.a Domain Controler and become Domain Admin.
