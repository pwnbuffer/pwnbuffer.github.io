1. Introduction Link to heading

EDR solutions a.k.a (Endpoint Detection and Response) are an evolution of the security software we know as antivirus. Modern EDRs use advanced techniques to detect threats, hooking APIs, isolating the host from the internet, analyzing behavior, and classifying detections based on Indicator of Compromise (IOC) rules classified by MITRE ATT&CK.

In this article, I will demonstrate how to bypass EDRs using evasive techniques.

2. Technical Analysis Link to heading

In a real attack, it is common to use “Loaders” and “Droppers” that load the main malware, going through stages. These stages can vary depending on the complexity of the malware, something we know as a Chain. This is necessary to prevent our attack from being interrupted by an AV/EDR running in the background.

int AESDecrypt(char* payload, unsigned int payload_len, char* key, SIZE_T keylen) {
    HCRYPTPROV hProv;
    HCRYPTHASH hHash;
    HCRYPTKEY hKey;

    if (!CryptAcquireContextW(&hProv, NULL, NULL, PROV_RSA_AES, CRYPT_VERIFYCONTEXT)) {
        return -1;
    }
    if (!CryptCreateHash(hProv, CALG_SHA_256, 0, 0, &hHash)) {
        return -1;
    }
    if (!CryptHashData(hHash, (BYTE*)key, (DWORD)keylen, 0)) {
        return -1;
    }
    if (!CryptDeriveKey(hProv, CALG_AES_256, hHash, 0, &hKey)) {
        return -1;
    }
    if (!CryptDecrypt(hKey, (HCRYPTHASH)NULL, 0, 0, payload, &payload_len)) {
        return -1;
    }
    CryptReleaseContext(hProv, 0);
    CryptDestroyHash(hHash);
    CryptDestroyKey(hKey);
    return 0;
}

We managed to bypass the AV/Defender, but it will still be detectable by an EDR. This happens due to the use of the VirtualAlloc, VirtualProtect, and CreateThread functions, which are commonly used in malware and, therefore, are intercepted by it using a technique called â€œAPI HOOKINGâ€.

API HOOKING Link to heading

A normal API call flow in Windows works as follows:

CreateRemoteThread() â†’ NtCreateThreadEx() â†’ SYSCALL

The CreateRemoteThread function calls NtCreateThreadEx hosted in ntdll.dll, which in turn calls the syscall. Many EDRs monitor NTAPI calls using trampoline hooks to check when a function is called by a program and what arguments it passes.

CreateRemoteThread() ---> NtCreateThreadEx() ----> SYSCALL 
                                   |      
                                   |       
                                   |      
                                EDR.DLL

We can ignore this by calling the syscall indirectly.
